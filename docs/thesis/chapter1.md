# บทที่ 1: บทนำ (Introduction)

---

## 1.1 ความเป็นมาและความสำคัญของปัญหา (Background and Problem Statement)

### 1.1.1 บริบทของมวยไท้เก๊กในยุคดิจิทัล

ในยุคดิจิทัลที่การเรียนรู้ด้วยตนเองผ่านสื่อออนไลน์ได้รับความนิยมสูง ศาสตร์การออกกำลังกายและศิลปะการต่อสู้ เช่น "มวยไท้เก๊ก" (Taijiquan) ได้รับความสนใจแพร่หลายและสามารถเข้าถึงผู้คนได้ง่ายขึ้นอย่างมาก มวยไท้เก๊กเป็นศิลปะการต่อสู้ภายใน (Internal Martial Arts) ที่มีรากฐานมายาวนานในประเทศจีน โดยเน้นการใช้ "เจตนา" นำ "พลังชี่" (Qi - Life Energy) และใช้ "พลังชี่" นำการเคลื่อนไหวของร่างกาย

> **หมายเหตุเรื่องคำศัพท์:** รายงานฉบับนี้ใช้คำว่า "Taijiquan" (ไท่จี๋เฉวียน) ซึ่งเป็นการถอดเสียงตามระบบ Pinyin จากภาษาจีน (太极拳) แทนคำว่า "Tai Chi" ที่นิยมใช้ในภาษาอังกฤษทั่วไป เพื่อความถูกต้องตามหลักวิชาการและสะท้อนถึงความหมายดั้งเดิมที่ครอบคลุมทั้ง "ไท่จี๋" (หลักปรัชญาหยิน-หยาง) และ "เฉวียน" (มวย/ศิลปะการต่อสู้)

**มวยไท้เก๊กสกุลเฉิน (Chen-style Taijiquan)** ถือเป็นต้นกำเนิดของมวยไท้เก๊กทุกสกุล มีลักษณะเด่นคือการเคลื่อนไหวที่ผสมผสานระหว่างความอ่อนนุ่มและความแข็งแกร่ง การปล่อยพลัง (ฟาจิ้ง) ที่รุนแรงรวดเร็ว และที่สำคัญที่สุดคือ **"ท่าม้วนไหม" (Silk Reeling / ฉานซือจิ้ง)** ซึ่งเป็นหัวใจสำคัญและเอกลักษณ์เฉพาะที่โดดเด่นจากมวยสกุลอื่นๆ

### 1.1.2 ความท้าทายของการเรียนรู้ด้วยตนเอง

อย่างไรก็ตาม การเข้าถึงมวยไท้เก๊กในยุคปัจจุบันมาพร้อมกับความท้าทายที่สำคัญ:

| ความท้าทาย | รายละเอียด |
|-----------|-----------|
| **ข้อจำกัดด้านเวลา** | ผู้ฝึกในวิถีชีวิตสมัยใหม่มีเวลาจำกัดในการเข้าเรียนกับครู |
| **ข้อจำกัดด้านสถานที่** | ไม่มีสถาบันสอนมวยไท้เก๊กในทุกพื้นที่ |
| **ข้อจำกัดด้านโอกาส** | การเข้าถึงครูผู้เชี่ยวชาญมีค่าใช้จ่ายสูงและหายาก |
| **การพึ่งพาสื่อออนไลน์** | ผู้สนใจจำนวนมากต้องเรียนรู้ผ่านวิดีโอและฝึกด้วยตนเอง |

**ปัญหาพื้นฐานที่เกิดขึ้นคือ:** การขาดความรู้ว่าการฝึกของตนเองถูกต้องหรือไม่ เพราะ **การขาดข้อมูลป้อนกลับ (Feedback)** ที่ถูกต้องและทันที ซึ่งเป็นหัวใจสำคัญของการพัฒนาทักษะที่ถูกต้องและปลอดภัย

### 1.1.3 ลักษณะพิเศษของท่าม้วนไหม

ท่าม้วนไหมของมวยไท้เก๊กสกุลเฉินมีลักษณะการเคลื่อนไหวที่ซับซ้อน ประกอบด้วย:

| ลักษณะ | คำอธิบาย |
|--------|---------|
| **การเคลื่อนไหวแบบเกลียวหมุน (Spiral Movement)** | ร่างกายเคลื่อนไหวเป็นเกลียวต่อเนื่อง ลื่นไหลไม่ติดขัด |
| **เปรียบเสมือนการสาวไหม** | เหมือนการสาวเส้นไหมออกจากรัง - ช้าเกินไปไหมไม่ออก เร็วเกินไปไหมขาด |
| **จุดกำเนิดพลังที่ตันเถียน** | พลังเริ่มจากแกนกลางลำตัว ถ่ายทอดผ่านเอวและสะโพก |
| **แสดงออกที่แขนขา** | พลังถูกส่งออกมาเป็นการเคลื่อนไหวที่ปลายมือ |
| **การถ่ายน้ำหนัก** | มีจังหวะการถ่ายน้ำหนักซ้าย-ขวาอย่างเป็นระบบ |

> **สิ่งสำคัญ:** คุณลักษณะเหล่านี้จำเป็นต้องประเมินผ่าน **"กระบวนการเคลื่อนไหว" (Movement Process)** ไม่ใช่เพียง "ท่าตำแหน่งสุดท้าย" (Static Pose)

### 1.1.4 ช่องว่างทางเทคโนโลยี

จากการทบทวนวรรณกรรมพบว่า เทคโนโลยีที่มีอยู่ในปัจจุบันมีข้อจำกัดดังนี้:

| ข้อจำกัด | ผลกระทบ |
|----------|---------|
| **เน้น Static Pose Matching** | ตรวจสอบเฉพาะตำแหน่งสุดท้ายของท่าทาง |
| **ขาด Movement Quality Analysis** | ไม่สามารถวิเคราะห์คุณภาพของกระบวนการเคลื่อนไหว |
| **Feedback ไม่สมบูรณ์** | ผู้ฝึกอาจได้รับข้อมูลป้อนกลับที่ไม่ครอบคลุม |
| **ความเสี่ยงระยะยาว** | อาจนำไปสู่การสร้างนิสัยการเคลื่อนไหวที่ผิดพลาดจนเกิดการบาดเจ็บเรื้อรัง |

### 1.1.5 แนวทางการแก้ปัญหา

ดังนั้น จากปัญหาและช่องว่างทางเทคโนโลยีที่กล่าวมา โครงการนี้จึงได้พัฒนา **"TaijiFlow AI"** ซึ่งเป็นเครื่องมือต้นแบบเพื่อ:

1. **วิเคราะห์ท่าทางแบบ Real-time** - ใช้เทคโนโลยี Pose Estimation (MediaPipe)
2. **ให้ข้อมูลป้อนกลับทันที** - Feedback ทั้งภาพและเสียง
3. **ประเมินคุณภาพการเคลื่อนไหว** - ใช้ Heuristics Engine ที่ออกแบบตามหลักการมวยไท้เก๊ก
4. **เข้าถึงได้ง่าย** - ทำงานบน Web Browser ใช้เพียง Webcam ไม่ต้องมีอุปกรณ์พิเศษ

## 1.2 วัตถุประสงค์ของโครงการ (Objectives)

โครงการ TaijiFlow AI มีวัตถุประสงค์หลัก 2 ประการ:

### 1.2.1 วัตถุประสงค์ที่ 1

**พัฒนาระบบปัญญาประดิษฐ์ต้นแบบ (Prototype AI System)** ที่สามารถวิเคราะห์และให้ข้อมูลป้อนกลับการเคลื่อนไหวของท่าม้วนไหมได้แบบทันที (Real-time)

| องค์ประกอบ | รายละเอียด |
|-----------|-----------|
| **แพลตฟอร์ม** | Web Application |
| **อุปกรณ์ที่ต้องการ** | Desktop/Laptop + Webcam |
| **เทคโนโลยี AI** | MediaPipe Pose Estimation |
| **การให้ Feedback** | Visual + Audio (TTS) |
| **ความเร็ว** | Real-time AI (Dynamic 5-10 FPS) / UI (30 FPS) |

### 1.2.2 วัตถุประสงค์ที่ 2

**สร้างชุดเมตริกเชิงปริมาณ (Quantitative Metrics)** สำหรับใช้ในการประเมินคุณภาพการเคลื่อนไหวตามหลักการของมวยไท้เก๊กใน 3 ระดับการฝึก

| ระดับ | ชื่อ | ลักษณะการฝึก |
|:-----:|-----|------------|
| L1 | ท่านั่ง (Sitting) | ขยับเฉพาะแขน |
| L2 | ท่ายืน (Standing) | ขยับลำตัวและแขน |
| L3 | ท่ายืนย่อ (Horse Stance) | ขยับทั้งตัว + ถ่ายน้ำหนัก |

**ชุดเมตริกประกอบด้วย 9 กฎ (Heuristics Rules):**

| # | กฎ | หลักการ |
|:-:|-----|---------|
| 1 | Path Accuracy | ตรวจสอบเส้นทางวงกลมและทิศทางหมุน |
| 2 | Arm Rotation | ตรวจสอบการหมุนแขน (หงาย/คว่ำ) |
| 3 | Elbow Sinking | ตรวจสอบการจมศอก |
| 4 | Waist Initiation | ตรวจสอบการใช้เอวนำ |
| 5 | Vertical Stability | ตรวจสอบความมั่นคงแนวดิ่ง |
| 6 | Smoothness | ตรวจสอบความลื่นไหล |
| 7 | Continuity | ตรวจสอบความต่อเนื่อง |
| 8 | Weight Shift | ตรวจสอบการถ่ายน้ำหนัก (L3 เท่านั้น) |

## 1.3 ขอบเขตของโครงการ (Scope)

### 1.3.1 ขอบเขตด้านเนื้อหามวยไท้เก๊ก

| รายการ | ขอบเขต | หมายเหตุ |
|--------|-------|---------|
| **สกุลมวย** | มวยไท้เก๊กสกุลเฉิน | ไม่รวมสกุลอื่นๆ (Yang Style, Wu Style, Sun Style, ...) |
| **ชุดท่า** | ท่าม้วนไหม (Silk Reeling) | ไม่รวมท่ารำมือ (Hand Form), ท่าอาวุธ (Weapon Form) |
| **ท่าย่อย** | ท่าม้วนไหมมือเดียว 4 รูปแบบ | ไม่รวมท่าม้วนไหมสองมือ (Double Hand Silk Reeling)|
| **ระดับการฝึก** | 3 ระดับ (นั่ง, ยืน, ยืนย่อ) | ไม่รวมท่าม้วนไหมเคลื่อนไหว (Movement Silk Reeling) |

**ท่าม้วนไหมมือเดียว 4 รูปแบบ:**

| # | ชื่อท่า | คำย่อ |
|:-:|--------|:-----:|
| 1 | มือขวาหมุนตามเข็ม | RH-CW |
| 2 | มือขวาหมุนทวนเข็ม | RH-CCW |
| 3 | มือซ้ายหมุนตามเข็ม | LH-CW |
| 4 | มือซ้ายหมุนทวนเข็ม | LH-CCW |

### 1.3.2 ขอบเขตด้านระบบซอฟต์แวร์

| รายการ | ขอบเขต |
|--------|-------|
| **แพลตฟอร์ม** | Web Application (Desktop/Laptop) |
| **อุปกรณ์** | เว็บแคมภายในเครื่องหรือภายนอก |
| **ฟังก์ชันหลัก** | Real-time Feedback ตามชุดเมตริก |
| **ฟังก์ชันเสริม** | ไม่รวมระบบจัดการผู้ใช้, บันทึกประวัติ |
| **AI Model** | ใช้ MediaPipe (Pre-trained) ไม่ฝึก Model ใหม่ |

### 1.3.3 ขอบเขตประชากร

**กลุ่มเป้าหมายหลัก:**

| เกณฑ์ | รายละเอียด |
|------|-----------|
| **ระดับ** | ผู้ฝึกระดับเริ่มต้นถึงระดับกลาง |
| **ความสนใจ** | ต้องการใช้เทคโนโลยีช่วยฝึกซ้อมด้วยตนเอง |
| **อุปกรณ์** | เข้าถึง Desktop/Laptop + Webcam ได้ |

**กลุ่มที่ไม่ใช่เป้าหมาย:**

- ผู้ฝึกมวยไท้เก๊กสกุลอื่น (หยาง, อู่, ซุน)
- ผู้ฝึกระดับสูง / ผู้เชี่ยวชาญ / ครูผู้สอน

### 1.3.4 ข้อจำกัดของอุปกรณ์ (Device Limitations)

ถึงแม้ระบบจะพัฒนาด้วย Web Technology ที่สามารถทำงานได้หลายแพลตฟอร์ม แต่เนื่องจากการประมวลผล AI (MediaPipe) ใช้ทรัพยากรเครื่องสูง:

- **Desktop/Laptop:** เป็นแพลตฟอร์มหลักที่แนะนำ (Primary Platform)
- **Tablet/iPad:** จัดอยู่ในกลุ่ม **"Supported with Adaptive Quality"** โดยระบบมี `PerformanceMonitor` ที่จะปรับลด Frame Rate และความละเอียดของการประมวลผล AI ให้เหมาะสมอัตโนมัติ (Lite Mode) เพื่อลดความร้อนและรักษาความลื่นไหล
- **Mobile Phone:** **"Supported with Warnings"** สามารถใช้งานได้แต่ไม่แนะนำเนื่องจากหน้าจอเล็ก โดยระบบจะแสดงคำเตือนก่อนเข้าใช้งาน

## 1.5 นิยามศัพท์ (Definition of Terms)

| ศัพท์ | นิยาม |
|------|------|
| **ระบบปัญญาประดิษฐ์ (AI System)** | ระบบคอมพิวเตอร์ที่ใช้กฎเชิง Heuristic และ Computer Vision ในการวิเคราะห์และประเมินท่าทาง |
| **Real-time Processing** | กระบวนการประมวลผลที่ให้ผลลัพธ์ตอบสนองทันทีในระหว่างการเคลื่อนไหว |
| **Pose Estimation** | เทคนิคที่ใช้ตรวจจับตำแหน่งข้อต่อจากภาพ/วิดีโอ สร้างเป็น Digital Skeleton |
| **Heuristics Engine** | กลไกที่ใช้กฎเกณฑ์เชิงตรรกะประเมินความถูกต้องของท่าทาง |
| **Quantitative Metrics** | ชุดกฎเกณฑ์และสูตรคำนวณที่แปลงหลักการมวยไท้เก๊กเป็นค่าที่วัดได้ |
| **Chen Style Taijiquan** | ศิลปะการต่อสู้ภายในของจีน ต้นกำเนิดของมวยไท้เก๊กทุกสกุล |
| **ท่าม้วนไหม (Silk Reeling)** | หลักการเคลื่อนไหวเป็นเกลียวต่อเนื่อง หัวใจของมวยไท้เก๊กสกุลเฉิน |
| **Adaptive Quality** | ระบบปรับคุณภาพกราฟิกและการประมวลผลอัตโนมัติตามประสิทธิภาพของอุปกรณ์ |

## 1.6 โครงสร้างของรายงาน (Report Structure)

รายงานการค้นคว้าอิสระฉบับนี้ประกอบด้วย 7 บท ดังนี้:

| บท | ชื่อบท | เนื้อหาหลัก |
|:--:|--------|-----------|
| **1** | บทนำ | ความเป็นมา, วัตถุประสงค์, ขอบเขต, ประโยชน์ |
| **2** | ทฤษฎีและงานวิจัยที่เกี่ยวข้อง | หลักการท่าม้วนไหม, Pose Estimation, งานวิจัยที่เกี่ยวข้อง |
| **3** | การวิเคราะห์ระบบ | Requirements, Use Cases, DFD, Activity Diagrams |
| **4** | การออกแบบระบบ | Architecture, Modules, Class/Sequence Diagrams |
| **5** | การพัฒนาระบบ | เครื่องมือ, Implementation, Core Features |
| **6** | การทดสอบระบบ | Unit/Functional/Usability Testing, Results |
| **7** | สรุปและข้อเสนอแนะ | Summary, Limitations, Future Work |
